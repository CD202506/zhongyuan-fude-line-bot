# 善信管理與團隊值勤規劃

## 文件目的

本文件補充 Web 後台 V2 的善信管理與團隊管理規劃。這是架構補充，不是正式上線功能。

本輪只修改文件：

- 不修改 `web_admin_mvp/` 畫面。
- 不修改正式 Google Sheets。
- 不修改 Render。
- 不修改 LINE Developers Webhook。
- 不修改正式 LINE Bot runtime。
- 不部署。
- 不 commit。

## 善信管理

### 目的

管理一般善信、發財金、平安龜、還金 / 還願、善信本人可查詢紀錄，並保留與帳務收入的關聯。

### 建議子模組

- 善信資料
- 發財金紀錄
- 平安龜紀錄
- 還金 / 還願紀錄
- 善信查詢紀錄
- 與帳務收入關聯

### 核心流程

```text
善信建檔
→ 登記求發財金 / 平安龜
→ 年度紀錄
→ 還金 / 還願
→ 關聯帳務收入
→ LINE Bot 可查詢本人授權內容
```

### 權限原則

- 善信本人只能查詢自己的授權紀錄。
- 善信可接收公開訊息，也可查詢本人授權內容。
- 團隊成員可協助查詢或登記部分資料，但不等於可管理帳務。
- 系統使用者依權限管理善信紀錄。
- 系統管理者維護分類、年度、權限與資料設定。

### 資料方向

未來可討論資料表：

- `devotees`
- `fortune_money_records`
- `peace_turtle_records`
- `repayment_records`
- `devotee_query_logs`

善信資料需特別注意授權、查詢範圍與個資保護。本階段不新增正式欄位、不寫入正式資料。

### 與帳務流水的關聯

善信管理中的以下紀錄可產生收入流水帳，並可被納入月度公告摘要：

- 發財金還金
- 平安龜還願
- 香油錢
- 捐獻
- 活動收入

善信本人只可查詢與自己相關的授權紀錄，不可查看完整內部帳務流水。

### 櫃檯接待窗口規劃

未來可規劃一個「櫃檯接待窗口」，讓現場人員輸入姓名或電話找到善信後，在同一畫面處理常見事項：

- 還發財金
- 還平安龜
- 添香油錢
- 補備註
- 選擇經手人 / 現場值班

此功能未來可由 adapter 分流至善信紀錄與帳務流水，但本階段不實作寫入、不接正式資料。

## 團隊管理

### 背景

廟方有輪班 / 值勤班表，例如「桃園中原福德宮委員服務輪值表」，欄位可包含姓名、輪值日期、備註、節日 / 農曆特殊日。

團隊管理要處理日常協作與值勤，不應混同於廟務職務任期或系統權限。

### 建議子模組

- 團隊成員
- 值勤班表
- 值勤週期
- 值勤提醒
- 節日 / 特殊日備註
- 與活動 / 來訪 / 公告關聯
- 與月度帳務公告對象關聯

### `duty_rosters` 資料方向

```text
duty_rosters
- roster_id
- member_id 或 team_member_id
- duty_start_date
- duty_end_date
- lunar_note
- festival_note
- note
- active
- created_at
- updated_at
```

### 權限與關聯原則

- 團隊成員可以不是系統使用者。
- 值勤班表不等於職務任期。
- 值勤人員可作為表單中的「經手人 / 現場值班」候選。
- 值勤提醒可與 LINE 通知或公告有關，但本階段不做正式推播。
- 值勤班表可關聯活動、來訪支援或公告提醒。
- 團隊成員可作為月度帳務公告的公告對象之一。
- 團隊成員可接收必要的月度公告或摘要，但不等於可管理完整帳務流水。
- 系統管理者可維護週期、提醒與資料設定。

## 與現有 Web 後台流程的關係

未來 Web 後台可新增：

- `善信管理`
- `團隊管理`

左側模組建議可變為：

```text
主控台
善信管理
友宮管理
來訪 / 請帖
公告 / 活動
團隊管理
帳務管理
管理者設定
```

本輪不修改 prototype 畫面，僅先記錄後續 IA 與資料模型方向。
