# 0.8.0A-8 廟務管理 / 採購 / 公文通知架構整理

本文件記錄 Web 後台 V2 的模組邊界整理。本輪只做文件化，不修改 `web_admin_mvp/` prototype、不接正式資料、不部署。

`0.8.0A-9` 補充 V1 freeze 決策：廟務管理、採購、公文 / 通知與帳務管理若未來正式化，不再以現有 Google Sheets 能否承擔作為設計限制。V1 只作為 legacy reference；V2 應依正確廟務流程、資料模型、權限、安全、API 與未來 LINE Bot 整合另行規劃資料核心。

## 核心命名

- 「廟務辦理」統一改稱「廟務管理」。
- 善信管理 = 管人。
- 廟務管理 = 管事。
- 帳務管理 = 管錢。

## 模組邊界

### 善信管理：管人

善信管理只負責善信基本資料與人員識別，不直接處理廟務事項本身，也不管理帳務科目。

建議欄位方向：

- 會員編碼
- 姓名
- 手機
- 電話
- 地址
- 出生日期
- 是否接受活動 / 廣告推送
- key in 人員代號
- 自動記錄 key in 日期時間
- 備註

善信管理可被廟務管理與帳務管理引用，例如發財金、平安龜、還金 / 還願、添油香等紀錄可關聯善信。

### 廟務管理：管事

廟務管理負責日常廟務事項，包含但不限於：

- 添油香
- 光明燈
- 太歲
- 發財金
- 平安龜
- 還金 / 還願
- 採購
- 公文 / 通知
- 廟務維持事項

廟務管理可以產生或關聯帳務流水，但不自行維護帳務科目主檔。

### 帳務管理：管錢

帳務管理負責收入、支出、帳務科目、金額、關聯來源、經手人與月報公告草稿。

命名統一：

- 帳務科目
- 費用科目
- 收支科目

以上三種說法在系統設計上應歸同一套帳務科目主檔管理，避免各模組自由輸入造成名稱混亂。

## 帳務科目主檔

- 帳務科目 / 費用科目 / 收支科目由帳務管理統一主檔管理。
- 善信管理、廟務管理、友宮管理、來訪 / 請帖、公告 / 活動、採購等模組只能選用既有科目。
- 其他模組不應自由輸入新科目。
- 每筆紀錄仍可保留備註欄位，用來記錄現場補充說明。
- 善信、廟務事項、友宮、來訪、活動、採購都可關聯帳務流水。

## 採購定位

採購是廟務管理的子功能。

採購管理負責：

- 品項
- 數量
- 用途
- 店家 / 供應商
- 經手人
- 關聯活動
- 關聯來訪 / 請帖
- 關聯廟務事項
- 備註

帳務管理負責：

- 付款
- 支出流水
- 帳務科目
- 金額
- 月報公告草稿是否納入

採購若產生支出，應由採購紀錄關聯到帳務流水。採購科目仍由帳務管理統一維護。

## 公文 / 通知定位

公文 / 通知先納入廟務管理，未來視規模再評估是否拆成獨立模組。

公文 / 通知可包含：

- 行政單位來文
- 行政單位發文
- 友宮來函
- 友宮回函
- 內部通知
- 會議通知
- 活動協調通知

公文 / 通知可關聯：

- 友宮
- 活動
- 來訪 / 請帖
- 採購
- 廟務事項

公文 / 通知可產生公告草稿，但不取代公告 / 活動。

## 與公告 / 活動的差異

- 公文 / 通知 = 文件紀錄。
- 公告 / 活動 = 發布內容或活動資料。
- 公文 / 通知可成為公告草稿的來源。
- 公告 / 活動可引用公文 / 通知，但不應把公文主資料直接混入公告主資料。

## 新增流程原則

- 新增友宮：先進友宮資料列表，再進新增 / 詳情流程。
- 新增來訪：先進來訪 / 請帖列表，再新增來訪紀錄；未來支援時間段，例如 `10:00-14:00`。
- 新增公告：先進公告 / 活動列表，再建立公告草稿；目前只做草稿 / 預覽 / 不正式發送。
- 新增帳務：先進帳務流水紀錄，再新增流水。
- 新增廟務事項：先進廟務管理列表，再新增對應事項，例如添油香、光明燈、太歲、發財金、平安龜、還金 / 還願。
- 新增採購：先進廟務管理中的採購列表，再新增採購紀錄；若有付款或支出，再關聯帳務流水。
- 新增公文 / 通知：先進廟務管理中的公文 / 通知列表，再建立文件紀錄；若需發布，再產生公告草稿。
