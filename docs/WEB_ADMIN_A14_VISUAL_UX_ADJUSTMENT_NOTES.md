# 0.8.0A-14 Visual MVP Review and UX Adjustment Notes

本文件記錄 A14 第一輪 `web_admin_app/` UI / UX 修正。本輪只調整前端視覺與互動基準，不串 API、不部署、不修改 V1、不修改 `web_admin_mvp/`。

## 1. 手機導覽

- 側邊導覽改為分組呈現：常用、日常作業、廟務文件。
- 手機 / 窄螢幕下導覽項目使用兩欄大按鈕，降低直向長度。
- 桌機維持左側單欄導覽，避免改變既有後台使用習慣。

## 2. 重要操作確認流程

- 詳情頁操作區改為「操作確認」。
- 操作包含：編輯資料、儲存草稿、送出確認、停用 / 封存。
- 不同模組會顯示更貼近情境的送出文字，例如「確認採購」、「確認回覆請帖」、「送出帳務確認」。
- 停用 / 封存等高風險操作僅管理者可用。
- 操作區提示改為一般使用者可理解文字，例如「送出前請先確認資料是否正確」。

## 3. 權限展示

- 新增角色切換：管理者、廟方人員、檢視者。
- 管理者可操作設定與高風險動作。
- 廟方人員可處理日常作業，但停用 / 封存與管理者設定需管理者確認。
- 檢視者以查看為主，新增、編輯與停用封存不可操作。

## 4. 模組欄位去樣板化

十個主要模組已加入各自的列表欄位與詳情欄位：

- 廟務管理：事項、負責、期限、待確認事項。
- 善信管理：服務、授權、最近紀錄、服務紀錄。
- 友宮管理：關係、窗口、最近互動、下一步。
- 來訪 / 請帖：型態、人數、請帖、接待窗口。
- 公告：對象、管道、文案狀態、關聯活動。
- 活動：日期、窗口、公告、準備事項。
- 採購管理：用途、驗收、帳務、請購原因。
- 公文 / 通知：類型、來源、處理、關聯廟務。
- 團隊管理：職務、值勤、權限、任期狀態。
- 帳務管理：科目、經手、月報、關聯來源。

## 5. 主控台調整

- 主控台保留日常總覽、近期活動、資料提醒、重要待辦與模組入口。
- 新增「重要待辦」區塊，讓待確認、待回覆、待整理、草稿項目更容易看到。
- 統計卡文字改為廟方日常用語，例如「採購待確認」、「待整理公文」、「帳務草稿」。
- 避免常用入口與模組入口過度重複，目前主控台以模組入口為主。

## 6. 保留原則

- 保留列表 → 詳情 → 詳情內編輯流程。
- 列表頁只瀏覽、搜尋、查看詳情。
- 新增入口維持單一清楚入口。
- 管理者設定集中於設定頁。
- 非管理者功能以 disabled 狀態與清楚文字呈現。

## 8. A14-Fix1 Role-based UI and CTA Behavior Adjustment

使用者人工檢查後，A14-Fix1 已修正：

- 角色導覽差異：只有管理者會在左側選單看到「管理者設定」；廟方人員與檢視者不顯示此入口。
- 詳情頁操作差異：管理者顯示完整操作確認；廟方人員顯示日常操作，高風險操作改為「需管理者確認」；檢視者不顯示操作按鈕，只顯示查看權限提示。
- CTA 前端提示：編輯資料、儲存草稿、送出確認、停用 / 封存點擊後會在卡片內顯示狀態訊息。
- 使用者畫面仍避免出現工程語言。

本修正仍未串 API、未部署、未修改 V1、未修改 `web_admin_mvp/`。

## 9. A14-Fix2 Permission Notice Placement and CTA Feedback Adjustment

使用者人工檢查後，A14-Fix2 已修正：

- 查看權限提示改為低干擾呈現：檢視者詳情頁不再顯示右側大型操作卡片，改在資料摘要上方顯示一行「查看模式」提示。
- CTA 點擊後改為明顯流程回饋：頁面會顯示狀態區塊，並依操作切換編輯、草稿、送出確認或停用 / 封存確認狀態。
- 編輯資料：資料摘要卡片進入編輯檢視，欄位呈現可編輯樣式，操作區改為儲存草稿、送出確認、取消編輯。
- 儲存草稿：頁面顯示「草稿已暫存」狀態，摘要狀態同步顯示草稿暫存。
- 送出確認：頁面顯示「已送出確認」狀態，提醒等待管理者或負責人確認。
- 停用 / 封存：管理者需先確認；廟方人員則顯示需管理者確認並建立確認事項。

本修正仍未串 API、未部署、未修改 V1、未修改 `web_admin_mvp/`。

## 10. A14-Fix3 Editable Field Interaction Adjustment

使用者人工檢查後，A14-Fix3 已修正：

- 編輯模式從單純視覺狀態提升為可操作欄位。
- 已加入文字輸入、textarea、日期、數字、下拉選單與多選標籤。
- 取消編輯會回復原資料顯示。
- 儲存草稿會保留目前前端暫存值，並顯示草稿摘要。
- 送出確認會顯示本次送出內容與確認流程提示。
- 檢視者仍不顯示操作 CTA，也無法進入編輯模式。

各模組欄位互動包含：

- 廟務管理：事項名稱、廟務類型、負責組別、日期、狀態、支援項目、備註。
- 善信管理：善信名稱、善信類型、服務紀錄、授權狀態、經手人、備註。
- 友宮管理：友宮名稱、地區、互動狀態、聯絡方式、關聯紀錄、下一步。
- 來訪 / 請帖：來訪類型、回覆狀態、來訪日期、關聯友宮、預計人數、備註。
- 公告：公告標題、發布管道、公告狀態、發布日期、內容摘要。
- 活動：活動類型、活動日期、支援項目、狀態、負責窗口。
- 採購管理：採購類別、金額、狀態、供應商、備註。
- 公文 / 通知：文件類型、公文日期、處理狀態、關聯活動或廟務、備註。
- 團隊管理：職務、權限角色、任期狀態、值勤安排、聯絡備註。
- 帳務管理：收支類型、金額、類別、關聯紀錄、備註。

本修正仍未串 API、未部署、未修改 V1、未修改 `web_admin_mvp/`。

## 11. A14-Fix4 Add New Record CTA Interaction

使用者人工檢查後，A14-Fix4 已修正所有模組列表頁「新增」CTA 無反應問題：

- 點擊新增後，會進入前端新增資料流程。
- 新增流程目前只作前端畫面流程，尚未串 API、未部署、未修改 V1。
- 表單包含儲存草稿、送出確認、取消新增。
- 儲存草稿會保留目前填寫內容並顯示草稿提示。
- 送出確認會顯示確認流程提示；採購、公文 / 通知、帳務等較敏感模組由廟方人員送出時，會提示需管理者確認後才會生效。
- 取消新增會回到列表。
- 檢視者不顯示可點擊新增 CTA，只顯示低干擾查看模式提示。

新增表單欄位已依模組分流，包含文字輸入、textarea、日期、數字、下拉選單與多選標籤。

## 12. A14-Fix5 List Search Add/Edit Flow and Archive Confirmation Adjustment

使用者人工檢查後，A14-Fix5 已修正列表、新增、編輯與封存流程：

- 模組主頁重新定義為：模組標題、單一新增 CTA、搜尋框、狀態篩選與列表。
- 新增流程已移出列表同層，改為各模組獨立 `/new` 頁面。
- 新增頁提供返回列表、模組專屬欄位、儲存草稿、送出確認與取消新增。
- 新增送出確認完成後回到模組主頁；取消新增也回到模組主頁。
- 編輯流程維持列表 → 詳情 → 詳情內編輯；列表頁不直接編輯、停用或封存。
- 儲存草稿、送出確認、停用 / 封存前加入頁面內確認小框框，包含標題、說明、取消與確認。
- 一般前端不提供刪除；高風險資料處理統一使用停用 / 封存，並保留紀錄。
- mock records 已加入 active / pending / archived / disabled / draft 狀態分類，列表可用前端篩選查看未封存、已封存或全部。
- 檢視者預設只能看到未停用 / 未封存資料，不顯示新增 CTA 與操作 CTA。

本修正仍未串 API、未部署、未修改 V1、未修改 `web_admin_mvp/`。
