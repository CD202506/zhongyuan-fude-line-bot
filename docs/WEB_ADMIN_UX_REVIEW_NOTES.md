# 0.8.0A-1 Web 後台 UX Review Notes

## 一、目前 Web 後台 MVP 狀態

- `web_admin_mvp/` 已作為第一版可視化 prototype。
- 目前僅使用 mock/dev data。
- 管委會 / 成員資料目前只作為少量示範資料或照片轉錄測試資料。
- 電話等資料需使用遮罩 / 假資料，避免 MVP prototype 暴露真實個資。
- 所有資料需標示為「開發測試資料 / 不代表正式名冊」。
- 不得寫入正式 Google Sheets 主檔 `中原福德宮_AppSheet_0612`。
- 不得影響正式 LINE Bot。
- 目前 prototype 未接正式 Google Sheets、未修改正式 LINE Bot runtime、未修改 Render 設定、未修改 LINE Developers Webhook、未部署。

## 二、已確認的 UX 方向

使用者已確認：

- 整體頁面視覺風格大致可接受。
- 目前主要問題不是風格，而是資訊架構、選單層次、文字密度與 CTA 配置。
- 後台應優先適合廟方人員、高齡使用者與手機 / 窄螢幕使用。

## 三、選單資訊架構最新決策

左側選單不應列出太多細項功能。

最新方向：左側選單只保留最上層模組：

1. 主控台
2. 友宮管理
3. 來訪 / 請帖
4. 公告 / 活動
5. 成員 / 職務
6. 財務管理
7. 管理者設定

細項功能不放在左側選單，而是放到各模組頁面內，以 CTA、卡片、分頁或區塊呈現。

例如：

- `新增 / 編輯友宮` 不放在左側選單，放在友宮管理頁面內。
- `友宮聯絡人` 不放在左側選單，放在友宮管理頁面內。
- `來訪型態設定` 不放在來訪一般操作中，移到管理者設定。
- `發布管道設定` 不放在公告 / 活動一般操作中，移到管理者設定。
- `收支分類` 不放在財務一般操作中，移到管理者設定。
- `最近查詢紀錄`、`查無資料紀錄`、`補資料建議 / 待補清單` 移到管理者設定下的 LINE / 查詢維運。

## 四、管理者設定最新決策

新增或保留父層：

```text
管理者設定
```

管理者設定內應集中：

### 1. 基礎設定

- 來訪型態設定
- 發布管道設定
- 職稱主檔
- 收支分類

### 2. 權限與角色

- 系統權限管理
- 管理者清單
- 職務任期設定
- `admin` 可轉移、增減、停用說明

### 3. LINE / 查詢維運

- 最近查詢紀錄
- 查無資料紀錄
- 補資料建議 / 待補清單
- LINE Bot 狀態說明

### 4. 系統與資料來源

- 開發資料來源說明
- AppSheet 備援說明
- 正式主檔保護提醒
- 不影響正式 LINE Bot 說明

管理者設定未來可依權限顯示：

- 無權限者可看到灰色 / 鎖定 / 需管理者權限提示，或未來隱藏。
- 有權限者可操作。
- 目前 prototype 不需要真的做登入或權限驗證。

## 五、應列入可視化範圍的模組

除了既有友宮、來訪、公告、成員之外，已決定納入可視化規劃：

- 活動管理
- 財務管理
- 查詢紀錄
- 查無資料紀錄
- 補資料建議 / 待補清單
- 管理者設定
- 權限與角色
- 系統與資料來源
- AppSheet 備援說明
- LINE Bot 狀態說明

這些目前只作為 prototype / mock / 佔位，不做正式功能、不寫入正式資料。

## 六、下一輪待修正 UX 重點

下一次使用者通知繼續時，優先修正：

1. 減少頁面文字量。
2. 移除大部分淺色說明文字。
3. 只保留標題與主要操作名稱。
4. 強化標題對比。
5. 重新整理 CTA 的位置與大小。
6. 避免同一區塊出現重複 CTA。
7. 卡片內不要塞太多灰色描述句。
8. 每個模組頁面以清楚 CTA / 卡片引導操作。
9. 手機 / 窄螢幕下避免選單過長。
10. 保持高齡使用者可讀性。

特別記錄：目前使用者針對「友宮管理」畫面指出：

- 淺色說明文字可取消。
- 只保留標題文字。
- 標題應用對比較強的方式呈現。
- CTA 位置與大小需重新調整。
- 避免頁面太多文字顯得擁擠。

## 七、全站資料管理流程

最新共通 UX 原則：

```text
先瀏覽 / 搜尋 → 查看詳情 → 詳情頁內編輯 / 管理關聯資料
```

套用規則：

- 列表頁負責瀏覽、搜尋、篩選與查看詳情。
- 每筆列表資料的主要操作以 `查看詳情` 為主。
- 列表頁不直接提供大量 `編輯`、`停用`、`刪除` 或關聯管理操作。
- 詳情頁才提供編輯、停用、刪除、管理聯絡人、管理關聯公告、管理權限或管理任期等操作。
- 新增 CTA 僅保留在明確少數位置，通常是模組頁右上角或主控台常用入口。
- 不在同一頁的列表與卡片中重複出現相同新增 CTA。
- 不讓快速通道、列表、相關資料區塊重複呈現同一批資料。

已套用到 prototype 的方向：

- 友宮列表只保留 `查看詳情`；編輯友宮、管理聯絡人、新增來訪紀錄、新增相關公告、停用友宮放在友宮詳情頁。
- 來訪 / 請帖列表只保留 `查看詳情`；編輯來訪紀錄、關聯友宮、新增相關公告、標記已回覆、作廢紀錄放在來訪詳情頁。
- 公告 / 活動列表只保留 `查看詳情`；編輯、管道設定、關聯來訪 / 活動、草稿 / 停用狀態放在詳情頁。
- 成員列表只保留 `查看詳情`；編輯成員、管理廟務職務、管理系統權限、管理任期、停用成員放在成員詳情頁。
- 財務列表只保留 `查看詳情`；編輯財務紀錄、儲存草稿、標記作廢、補備註、類別檢視放在財務詳情頁。

## 八、主控台與管理者設定分工

- 左側選單只保留 7 個頂層模組：主控台、友宮管理、來訪 / 請帖、公告 / 活動、成員 / 職務、財務管理、管理者設定。
- 主控台不混合不同層級功能：日常總覽、常用入口、近期活動、管理者資訊需分開呈現。
- 管理者設定集中放設定、權限、LINE 維運與資料來源。
- `來訪型態設定`、`發布管道設定`、`職稱主檔`、`收支分類` 放在管理者設定。
- `最近查詢紀錄`、`查無資料紀錄`、`補資料建議 / 待補清單` 放在管理者設定下的 LINE / 查詢維運。
