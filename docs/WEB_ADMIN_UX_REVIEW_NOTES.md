## 二、已確認的 UX 方向

使用者已確認：

- 整體頁面視覺風格大致可接受。
- 目前主要問題不是風格，而是資訊架構、選單層次、文字密度與 CTA 配置。
- 後台應優先適合廟方人員、高齡使用者與手機 / 窄螢幕使用。

## 三、選單資訊架構最新決策

左側選單不應列出太多細項功能。

最新方向：左側選單只保留最上層模組：

1. 主控台
2. 友宮管理
3. 來訪 / 請帖
4. 公告 / 活動
5. 團隊管理
6. 帳務管理
7. 管理者設定

細項功能不放在左側選單。一般模組主頁以列表、搜尋、篩選與查看詳情為主；細項操作放到詳情頁或管理者設定。

例如：

- `新增 / 編輯友宮` 不放在左側選單，放在友宮管理頁面內。
- `友宮聯絡人` 不放在左側選單，放在友宮詳情頁或管理者進階流程中。
- `來訪型態設定` 不放在來訪一般操作中，移到管理者設定。
- `發布管道設定` 不放在公告 / 活動一般操作中，移到管理者設定。
- `收支分類` 不放在帳務一般操作中，移到管理者設定。
- `最近查詢紀錄`、`查無資料紀錄`、`補資料建議 / 待補清單` 移到管理者設定下的系統操作紀錄。

## 四、管理者設定最新決策

新增或保留父層：

```text
管理者設定
```

管理者設定內應集中：

### 1. 標籤 / 主檔管理

- 發布管道
- 發布狀態
- 來訪型態
- 廟務職務
- 友宮聯絡人職稱
- 系統權限
- 帳務分類

### 2. 權限與角色

- 系統權限管理
- 管理者清單
- 權限規則說明
- `admin` 可轉移、增減、停用說明

具體職務任期、值勤班表與人員資料歸在團隊管理，不放在管理者設定主內容。

### 3. 系統操作紀錄

- 最近查詢紀錄
- 查無資料紀錄
- 補資料建議 / 待補清單
- LINE Bot 狀態說明

### 4. 系統與資料來源

- 開發資料來源說明
- AppSheet 備援說明
- 正式主檔保護提醒
- 不影響正式 LINE Bot 說明

管理者設定未來可依權限顯示：

- 無權限者可看到灰色 / 鎖定 / 需管理者權限提示，或未來隱藏。
- 有權限者可操作。
- 目前 prototype 不需要真的做登入或權限驗證。

## 七、全站資料管理流程

最新共通 UX 原則：

```text
先瀏覽 / 搜尋 → 查看詳情 → 詳情頁內編輯 / 管理關聯資料
```

套用規則：

- 列表頁負責瀏覽、搜尋、篩選與查看詳情。
- 每筆列表資料的主要操作以 `查看詳情` 為主。
- 列表頁不直接提供大量 `編輯`、`停用`、`刪除` 或關聯管理操作。
- 詳情頁才提供編輯、停用、刪除、管理聯絡人、管理關聯公告、管理權限或管理任期等操作。
- 新增 CTA 僅保留在明確少數位置，通常是模組頁右上角或主控台常用入口。
- 不在同一頁的列表與卡片中重複出現相同新增 CTA。
- 不讓快速通道、列表、相關資料區塊重複呈現同一批資料。
- 各模組主頁不優先顯示大量快速卡片；若卡片只是重複列表功能，應移除。
- 設定類、權限類、系統操作紀錄、資料來源類入口集中到管理者設定，不放在一般模組主頁。

模組語意邊界：

- 善信管理 = 管人。
- 廟務管理 = 管事。
- 帳務管理 = 管錢。
- 友宮 = 宮廟主檔。
- 來訪 / 請帖 = 友宮互動事件。
- 公告 / 活動 = 發布或活動內容。
- 近期廟務動態 = 對人可讀的廟務事件，不顯示系統 log。
- 系統操作紀錄 = 管理者維運資訊。

已套用到 prototype 的方向：

- 友宮管理主頁以友宮資料列表為主，上方快速卡片移除；列表只保留 `查看詳情`；編輯友宮、管理聯絡人、新增來訪紀錄、新增相關公告、停用友宮放在友宮詳情頁。
- 來訪 / 請帖主頁以來訪 / 請帖紀錄列表為主，上方快速卡片移除；列表只保留 `查看詳情`；編輯來訪紀錄、關聯友宮、新增相關公告、標記已回覆、作廢紀錄放在來訪詳情頁。
- 公告 / 活動主頁以公告列表與活動列表為主，上方快速卡片移除；列表只保留 `查看詳情`；編輯、管道欄位設定、關聯來訪 / 活動、草稿 / 停用狀態放在詳情頁。
- 團隊管理主頁以團隊成員和值勤班表列表為主；廟務職務 / 系統權限是下一層內容，不放在左側選單。列表只保留 `查看詳情`；編輯、管理職務、管理系統權限、管理任期、停用放在詳情頁。
- 帳務管理主頁以帳務流水紀錄列表為主，上方快速卡片移除；列表只保留 `查看詳情`；編輯帳務流水紀錄、儲存草稿、標記作廢、補備註、類別檢視放在帳務詳情頁。

## 八、關聯資料呈現原則

詳情頁中的關聯資料只能作為摘要，不應混成主資料本身。

- 友宮詳情頁：聯絡人、來訪 / 請帖、相關公告只顯示數量與最近摘要，提供查看或新增 CTA。
- 來訪詳情頁：關聯公告 / 活動只顯示摘要，需進入公告或活動詳情後再編輯。
- 公告詳情頁：關聯來訪 / 活動只顯示摘要，需進入對應詳情後再編輯。
- 成員詳情頁：職務任期 / 權限只顯示摘要與說明，管理操作放在詳情頁 CTA 或管理者設定。
- 帳務詳情頁：類別、草稿、作廢狀態只顯示摘要與操作入口，不在列表中直接編輯。

## 九、主控台與管理者設定分工

- 左側選單只保留頂層模組：主控台、善信管理、友宮管理、來訪 / 請帖、公告 / 活動、團隊管理、帳務管理、管理者設定。
- 主控台維持簡潔：日常總覽、常用入口、近期廟務動態、管理者資訊分開呈現。
- 主控台不放成員測試資料、帳務草稿、mock/dev data、查無資料、待補資料或系統狀態大卡片。
- 管理者設定集中放設定、權限、系統操作紀錄與資料來源。
- `來訪型態設定`、`發布管道設定`、`職稱主檔`、`收支分類` 放在管理者設定。
- `最近查詢紀錄`、`查無資料紀錄`、`補資料建議 / 待補清單` 放在管理者設定下的系統操作紀錄。

## 十、第三方驗證前 UX cleanup

`0.8.0A-4 Internal UX Cleanup Before Third-party Preview` 先處理以下問題：

- 次頁、詳情頁與表單頁都需提供返回 CTA。
- 表單頁需提供 `返回上一頁`，若有來源資料也需提供 `返回來源資料`。
- 管理者設定子頁需提供 `返回管理者設定`。
- 管理者設定不放具體人員任期資料；職務任期屬於團隊管理。
- 標籤 / 主檔管理採分類化：
  - 發布管道：LINE、Facebook、VOOM、宮廟公告欄。
  - 發布狀態：草稿、預覽、不正式發送、已發布。
  - 來訪型態：參訪、用餐、請帖、祝壽、進香、其他。
  - 廟務職務：主任委員、總幹事、值年爐主、值年副爐主、委員。
  - 友宮聯絡人職稱：主委、總務、聯絡窗口、秘書。
  - 系統權限：admin、staff、viewer。
  - 帳務分類：香油錢、發財金還金、平安龜還願、餐費、供品、其他。
- `LINE / Facebook / VOOM` 是發布管道；`不正式發送` 是發布狀態或安全提醒。
- `admin / staff / viewer` 是系統權限，不是廟務職務。
- 主要列表頁搜尋方式需一致：善信、友宮、來訪 / 請帖、公告 / 活動、帳務管理優先使用單一模糊搜尋列。
- 下拉選單可保留在新增 / 編輯表單或管理者設定，不作為一般列表頁主要搜尋方式。
- 列表頁不放 `寫入狀態`、`資料說明`、敏感資訊提示等開發欄位；安全邊界改由全站提示、管理者設定與文件記錄。

## 十一、資訊架構 cleanup

`0.8.0A-6 Information Architecture Cleanup Before Third-party Preview` 先處理以下問題：

- 主控台 `近期廟務動態` 只放對人可讀的廟務事件，例如來訪、公告、活動、善信服務、團隊值勤提醒。
- 主控台不放 `友宮資料更新`、`帳務流水`、`善信紀錄`、`查無資料`、`補資料建議`、`LINE 查詢紀錄` 或測試資料狀態。
- 系統 log 與操作紀錄歸管理者設定的 `系統操作紀錄`。
- 友宮名稱欄只顯示宮廟主檔名稱，不混入 `參訪`、`祝壽`、`請帖`、`會香` 等事件描述。
- 友宮最近互動可放在獨立欄位或摘要。
- 來訪 / 請帖是友宮互動事件；公告 / 活動是發布或活動內容。
- 來訪 / 請帖可以產生公告，但公告不是來訪主資料，也不取代來訪紀錄。

## 十二、預覽前 UI residue cleanup

`0.8.0A-7 Preview UI Residue Cleanup` 先處理以下問題：

- 來訪 / 請帖列表的 `友宮名稱` 欄只顯示宮廟主檔名稱。
- 來訪 / 請帖的事件描述放在獨立 `主題` 欄位，不與友宮名稱上下堆疊。
- 一般來訪 / 請帖列表頁不顯示 `來訪型態顯示` 或型態主檔清單。
- 來訪型態主檔只出現在管理者設定的標籤 / 主檔管理中。
- 團隊管理主頁只保留團隊成員列表與值勤班表。
- `廟務職務 / 系統權限` 快速卡片不放在團隊管理主頁。
- 系統權限設定歸管理者設定；查看個人職務、任期與權限放在團隊成員詳情頁。

## 十三、廟務管理 / 採購 / 公文通知架構

`0.8.0A-8 Temple Affairs / Procurement / Document Architecture Review` 只做文件整理：

- 「廟務辦理」改名為「廟務管理」。
- 善信管理只管人：會員編碼、手機、電話、地址、出生日期、是否接受活動 / 廣告推送、key in 人員代號、自動記錄 key in 日期時間。
- 廟務管理管日常廟務事項：添油香、光明燈、太歲、發財金、平安龜、還金 / 還願、採購、公文 / 通知、廟務維持。
- 帳務管理管錢：收入、支出、帳務科目、金額、關聯來源、經手人、月報公告草稿。
- 帳務科目 / 費用科目 / 收支科目由帳務管理統一主檔管理；其他模組只能選用既有科目，不應自由輸入。
- 採購是廟務管理子功能；付款與支出流水關聯帳務管理。
- 公文 / 通知先歸廟務管理，可包含行政單位來文 / 發文、友宮來函 / 回函、內部通知、會議通知、活動協調通知。
- 公文 / 通知不等於公告 / 活動；公文是文件紀錄，公告 / 活動是發布內容或活動資料。
- 首頁未來可增加單純主選項頁面；左側選單未來可自動隱藏 / 點選展開。
- 未來可規劃行事曆呈現來訪、帳務、活動、值勤；友宮詳情未來可規劃地圖呈現。
