## 4. A13 與 `web_admin_mvp/` 的主要差異

### 視覺差異

- `web_admin_mvp/` 是靜態 prototype，資訊較多，保留較多測試與安全說明。
- A13 使用 Vite React 重建，視覺更接近正式後台基礎，卡片、狀態標籤、按鈕與響應式規則較一致。
- A13R 前已移除頁面中較工程化的 mock / API / 資料庫提示，使畫面更像給廟方 review 的管理介面。

### 導覽差異

- `web_admin_mvp/` 左側選單以 8 個入口為主，公告 / 活動仍合併。
- A13 依最後模組邊界拆出更多正式模組入口：廟務管理、採購管理、公文 / 通知、公告、活動皆成為一級入口。
- A13 導覽更完整，但手機上可能更長，A14 需要檢查是否需分組或收合。

### 主控台差異

- `web_admin_mvp/` 主控台偏 prototype 操作入口與近期廟務動態。
- A13 主控台加入較完整日常總覽卡片，覆蓋友宮、來訪、請帖、公告、活動、採購、公文、帳務。
- A13 有模組入口卡片與近期活動列表，但常用入口與模組入口的分工仍需 A14 檢查。

### 模組入口差異

- `web_admin_mvp/` 早期為流程與模組討論服務，部分入口曾作為快速卡片或子功能入口。
- A13 模組入口更接近正式 V2 模組邊界，採統一 `ModuleCard` 呈現。
- A13 尚未為每個模組定製不同列表欄位，仍使用通用樣板。

### 列表 / 詳情流程差異

- `web_admin_mvp/` 已確立列表 → 詳情 → 詳情內操作原則，且不同模組有較多 prototype 內容。
- A13 已承接流程原則，但列表與詳情是共用樣板，尚未呈現各模組完整欄位差異。
- A13 詳情頁操作區比較乾淨，但二次確認流程尚未設計。

### 管理者設定差異

- `web_admin_mvp/` 管理者設定包含基礎設定、權限、系統操作紀錄、資料來源與 LINE 維運說明。
- A13 管理者設定先保留高層分類：來訪型態、發布管道、帳務科目、權限角色、資料狀態、LINE Bot 整合設定。
- A13 尚未呈現完整的系統操作紀錄、查無資料、補資料建議等管理者維運內容。

### 使用者可能會感受到的差異

- A13 更像正式後台畫面，視覺一致度較高。
- A13 模組更多，初次使用可能需要更清楚的分組與導覽提示。
- A13 的操作比較簡潔，但部分模組內容仍像樣板，廟方可能需要更多具體欄位才好判斷。

### 只有工程層才會感受到的差異

- A13 是 Vite + React + TypeScript 專案，與 `web_admin_mvp/` 的靜態 HTML / JS prototype 分離。
- A13 使用共用資料設定、共用 route、共用列表 / 詳情頁，後續可逐步接 API。
- A13 尚未建立 API contract、正式登入、正式權限或資料寫入流程。

## 6. A14 優先修正清單

### A. 必須修正

- 檢查手機版側邊導覽是否過長，必要時改成分組、抽屜或底部入口。
- 為重要操作補足確認流程設計，例如停用、作廢、儲存草稿、管理者設定變更。
- 檢查各模組列表 / 詳情是否需要各自不同欄位，避免所有模組看起來過於樣板化。
- 補強 staff / viewer 權限畫面展示，確認非管理者不會誤觸管理者功能。

### B. 建議修正

- 主控台常用入口與模組入口的分工再整理，避免功能重複。
- 精簡模組描述文字，降低年長使用者閱讀負擔。
- 補充 LINE Bot 整合入口的廟方語彙說明，避免工程詞彙。
- 將管理者設定分區再整理為「基礎設定、權限角色、資料狀態、LINE Bot 整合」等更清楚群組。

### C. 可延後

- 建立 API contract 與 mock API 層。
- 正式登入、正式權限、audit log 與資料驗證。
- 行事曆、地圖、月報公告草稿細節。
- 各模組完整新增 / 編輯表單。

### D. 不建議處理

- 不建議把 `web_admin_mvp/` 直接改成正式前端。
- 不建議在 A14 前急著串 Render API 或 PostgreSQL。
- 不建議把 V1 Google Sheets / LINE Bot 現有 runtime 納入 A13 / A14 前端調整。
- 不建議在列表頁加入直接編輯、刪除或大量快速操作。

## 7. 本輪不做事項
