# Web 後台 V2 角色與模組架構補充

## 文件目的

本文件補充 Web 後台 V2 的人員層級、權限概念與模組邊界。這是架構規劃文件，不是正式實作計畫。

本輪只補強文件：

- 不修改正式 LINE Bot runtime。
- 不部署。
- 不 commit。

## 人員層級

| 層級 | 來源 | 系統關係 | 權限 | 相關模組 |
| --- | --- | --- | --- | --- |
| 任何人 | LINE 官方帳號外部使用者 | 尚未成為好友或尚未建檔 | 無 | LINE |
| 善信 | LINE 官方帳號好友、現場登記、求發財金 / 平安龜、還金 / 還願者 | 善信資料、LINE Bot 查詢 | 接收公開訊息、查詢本人授權內容；不可查詢內部帳務流水 | 善信管理、活動、公告、發財金、平安龜、還金 / 還願 |
| 團隊成員 | 志工、現場協助者、值勤人員 | 值勤、協作、通知 | 同善信，並可接收值勤、公告、活動、來訪支援等必要通知；不一定有帳務管理權限 | 團隊管理、班表、活動、公告、來訪支援 |
| 系統使用者 | 管理委員會成員 | 後台操作 | 依權限模組 | 來訪、活動、公告、友宮、善信、帳務 |
| 系統管理者 | 管理委員會成員 | 系統維運 | 最高權限 | 全模組、系統設定、權限、資料來源、LINE 維運 |

## 角色邊界

- 團隊成員不一定等於系統使用者。
- 管理委員會成員不一定永遠是系統管理者。
- `admin` 是系統權限，不是廟務職務。
- 廟務職務、團隊值勤、系統權限必須分開。
- 善信可以成為團隊成員，但不因此自動取得後台操作權限。
- 系統使用者可以操作後台，但不一定具備系統管理者權限。
- 系統管理者權限應可轉移、增減、停用，不應綁定單一個人或單一廟務職稱。
- 宮廟現場公用電腦情境下，表單不可只依賴登入 session 推定經手人，需有顯性的「經手人 / 現場值班」欄位。

## 權限概念

### 廟務職務

用來描述人在廟務組織中的職稱與任期，例如：

- 主任委員
- 總幹事
- 值年爐主
- 值年副爐主
- 委員
- 監事

### 團隊值勤

用來描述協助廟務的班表、輪值、提醒與支援關係，例如：

- 值勤日期
- 值勤週期
- 節日 / 特殊日備註
- 來訪支援
- 活動支援

值勤班表不等於廟務職務任期，也不等於系統權限。

### 系統權限

用來描述後台或系統中可執行的操作等級，例如：

- `admin`
- `staff`
- `viewer`

`admin` 不等於總幹事。使用者本人目前可作為「總幹事 / admin」範例，但未來 `admin` 必須可轉移、增減、停用。

## Web 後台模組建議

未來 Web 後台左側模組可調整為：

```text
主控台
善信管理
友宮管理
來訪 / 請帖
公告 / 活動
團隊管理
帳務管理
管理者設定
```

廟務職務與系統權限是團隊管理的下一層內容，不放在左側選單。

## 模組定位

### 善信管理

管理一般善信、發財金、平安龜、還金 / 還願、善信本人可查詢紀錄，以及與帳務收入的關聯。

### 團隊管理

管理團隊成員、值勤班表、值勤週期、值勤提醒，以及與活動、來訪、公告的關聯。

### 團隊管理中的職務 / 權限

管理廟務職務、職務任期、管委會成員與系統使用者的關係。

### 管理者設定

集中設定類、權限類、LINE 維運類與資料來源類內容。

### 帳務管理

未來正式模組名稱應使用「帳務管理」，功能定位是廟務流水帳紀錄、收支摘要與月報公告草稿，不是完整會計、報稅、審計或銀行帳務平台。

帳務管理需支援：

- 帳務流水紀錄。
- 收支流水帳。
- 月報公告草稿。
- 對指定對象的 LINE 群組公告內容輔助。

月度公告對象至少包含：

- 團隊成員。
- 管理委員會成員。
- 系統管理者。

善信本人只可查詢與自己相關的授權資料，不可查看完整內部帳務流水。
